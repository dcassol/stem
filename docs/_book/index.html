Welcome

About

This repository containers RNA-seq pipeline, docker container and sample data for testing. Pipelines are written in WDL language and Cromwell workflow execution engine is used to run them.

Is this workflow useful for you?

Expected output

  • Fastqc Quality Report
  • Reads Trimming with Trimmomatic
  • Align Reads to Reference Genome using HISAT2 (BAM files)
  • Expression quantification using summarizeOverlaps (GenomicFeatures) (Lawrence et al. 2013)
  • Differential Expression Matrix using DESeq2
  • Create Feature set and Filtered Differential Expression Matrix

Quick Start - No Installation required 

docker run  -dP dcassol/wdl_rnaseq
docker ps 
docker exec -it <IMAGE NAME> /bin/bash
cromwell run rnaseq.wdl -i rnaseq.json -o rnaseq.options.json

Now, you can test and edit the pipeline for your data. For more details, please check the Sample Data section. Have fun!

Quick Start

Step 1: Clone the git repository to a directory on your computer:

https://github.com/dcassol/wdl_rnaseq.git

Note: Ensure you have the correct permissions to run the pipeline - here we are using Docker. A further section will be provided containing more details if you are running in an HPC environment.

Step 2: To run the pipeline use the command below:

cromwell run rnaseq.wdl -i rnaseq.json -o rnaseq.options.json 
## OR
java -jar cromwell-72.jar run rnaseq.wdl -i rnaseq.json -o rnaseq.options.json

Please, make sure use the correct PATH to cromwell-72.jar file.

0.1 Import csv 

occupation_by_state <- read.csv("data/occupation.csv")
head(occupation_by_state)
filtered_data <- occupation_by_state %>%
  filter(grepl("B24020_019|B24020_170", variable, ignore.case = TRUE))

proportions <- filtered_data %>%
  group_by(Year, NAME, sex) %>%  # Assuming 'NAME' is the state name
  summarise(total_estimate = sum(estimate, na.rm = TRUE), .groups = 'drop') %>%
  group_by(Year, NAME) %>%
  mutate(proportion = total_estimate / sum(total_estimate)) %>%
  select(Year, NAME, sex, proportion)


proportions <- proportions %>%
  mutate(proportion = proportion * 100) 
# Library for interactive visualization
# Library for interactive visualization
library(plotly)

# Get unique states
state_levels <- unique(proportions$NAME)

# Prepare the buttons for the dropdown menu for state selection
buttons <- lapply(state_levels, function(state) {
  list(method = "update",
       args = list(list(visible = sapply(proportions$NAME, function(x) x == state)),
                   list(title = paste("Proportion of Male/Female in", state, "over the Years"))),
       label = state)
})

# Create the plot with all traces initially visible but will toggle with dropdown
plot <- plot_ly()

# Loop through each state to add traces for Male and Female
for (state in state_levels) {
  plot <- plot %>%
    add_trace(data = proportions[proportions$NAME == state & proportions$sex == "Male", ],
              x = ~Year, y = ~proportion, mode = 'lines+markers',
              name = paste("Male", state), visible = state == state_levels[1],  # Visibility set for the first state
              line = list(color = 'blue', width = 4)) %>%
    add_trace(data = proportions[proportions$NAME == state & proportions$sex == "Female", ],
              x = ~Year, y = ~proportion, type = 'scatter', mode = 'lines+markers',
              name = paste("Female", state), visible = state == state_levels[1],  # Visibility set for the first state
              line = list(color = 'red'), marker = list(size = 7))
}

# Final layout configuration
plot <- plot %>%
  layout(title = "Proportion of Male/Female over the Years by State",
         xaxis = list(title = 'Year'),
         yaxis = list(title = 'Percentage', range = c(0, 100)),
         hovermode = 'closest',
         showlegend = TRUE,
         updatemenus = list(
           list(
             x = 0.0,
             xanchor = 'left',
             y = 1.0,
             yanchor = 'top',
             buttons = buttons
           )
         ))

# Render the plot
